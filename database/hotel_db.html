To start your Hotel Booking System in MySQL, we’ll define the three main tables as described:

✅ 1. rooms Table

Stores all rooms in the hotel with type, price, and availability.

sql
CREATE TABLE rooms (
    room_id VARCHAR(10) PRIMARY KEY,  -- e.g. '1A', '10C'
    floor CHAR(1),                    -- 'A', 'B', or 'C'
    room_number INT,                 -- 1 to 10
    room_type ENUM('Basic', 'Deluxe', 'Suite'),
    price DECIMAL(8,2),
    is_available BOOLEAN DEFAULT TRUE
);

✅ 2. customers Table

Stores customer account information.

sql
CREATE TABLE customers (
    customer_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    age INT,
    gender ENUM('Male', 'Female', 'Other'),
    password VARCHAR(255) NOT NULL
);

✅ You can use Java Swing with secure hashed passwords in real apps.

✅ 3. bookings Table

Stores booking details, linked to rooms and customers.

sql
CREATE TABLE bookings (
    booking_id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT,
    room_id VARCHAR(10),
    check_in_date DATE,
    check_out_date DATE,
    
    guests_0_6 INT DEFAULT 0,
    guests_7_14 INT DEFAULT 0,
    guests_15_20 INT DEFAULT 0,
    guests_20_plus INT DEFAULT 0,
    
    total_guests INT,
    booking_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (customer_id) REFERENCES customers(customer_id),
    FOREIGN KEY (room_id) REFERENCES rooms(room_id)
);
